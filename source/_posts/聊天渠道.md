<h2>聊天渠道</h2><p>OpenClaw可以在你已经使用的任何聊天应用上与你交流。每个渠道通过Gateway网关连接。所有渠道都支持文本；媒体和表情回应的支持因渠道而异。</p><h2>支持的渠道</h2><ul><li><strong>BlueBubbles</strong> — 推荐用于iMessage；使用BlueBubbles macOS服务器REST API</li><li><strong>Discord</strong> — Discord Bot API + Gateway；支持服务器、频道和私信</li><li><strong>飞书</strong> — 飞书（Lark）机器人（插件，需单独安装）</li><li><strong>Google Chat</strong> — 通过HTTP webhook的Google Chat API应用</li><li><strong>iMessage（旧版）</strong> — 通过imsg CLI的旧版macOS集成</li><li><strong>LINE</strong> — LINE Messaging API机器人（插件）</li><li><strong>Matrix</strong> — Matrix协议（插件）</li><li><strong>Mattermost</strong> — Bot API + WebSocket（插件）</li><li><strong>Microsoft Teams</strong> — Bot Framework（插件）</li><li><strong>Nextcloud Talk</strong> — 通过Nextcloud Talk的自托管聊天（插件）</li><li><strong>Nostr</strong> — 通过NIP-04的去中心化私信（插件）</li><li><strong>Signal</strong> — signal-cli；注重隐私</li><li><strong>Slack</strong> — Bolt SDK；工作区应用</li><li><strong>Telegram</strong> — 通过grammY使用Bot API；支持群组</li><li><strong>Tlon</strong> — 基于Urbit的消息应用（插件）</li><li><strong>Twitch</strong> — 通过IRC连接的Twitch聊天（插件）</li><li><strong>WebChat</strong> — 基于WebSocket的Gateway网关WebChat界面</li><li><strong>WhatsApp</strong> — 最受欢迎；使用Baileys，需要二维码配对</li><li><strong>Zalo</strong> — Zalo Bot API（插件）</li><li><strong>Zalo Personal</strong> — 通过二维码登录的Zalo个人账号（插件）</li></ul><h2>注意事项</h2><ul><li>渠道可以同时运行；配置多个渠道后，OpenClaw会按聊天进行路由</li><li>最快的设置方式通常是Telegram（简单的机器人令牌）</li><li>群组行为因渠道而异</li><li>为安全起见，私信配对和允许列表会被强制执行</li></ul>
