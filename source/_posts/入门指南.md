<h2>目标：尽快从零到第一个可用聊天</h2><p>最快聊天：打开Control UI（无需渠道设置）。运行openclaw dashboard并在浏览器中聊天，或在Gateway网关上打开http://127.0.0.1:18789/</p><h2>推荐路径：使用CLI新手引导向导</h2><p>运行openclaw onboard --install-daemon，它会设置：模型/认证（推荐OAuth）、Gateway网关设置、渠道（WhatsApp/Telegram/Discord/Mattermost等）、配对默认值（安全私信）、工作区引导+Skills、可选的后台服务</p><h2>前置条件</h2><ul><li>Node.js >= 22</li><li>pnpm（可选；从源代码构建则推荐）</li><li>Brave Search API密钥用于网页搜索</li></ul><h2>安装CLI</h2><p>curl -fsSL https://openclaw.ai/install.sh | bash</p><h2>Windows (PowerShell)</h2><p>iwr -useb https://openclaw.ai/install.ps1 | iex</p><h2>替代方案（全局安装）</h2><p>npm install -g openclaw@latest 或 pnpm add -g openclaw@latest</p><h2>运行新手引导向导</h2><p>openclaw onboard --install-daemon</p><h2>凭证存储位置</h2><p>推荐的Anthropic路径：设置API密钥</p><h2>启动Gateway网关</h2><p>如果安装了服务，Gateway网关应该已经在运行：openclaw gateway status</p><p>手动运行（前台）：openclaw gateway --port 18789 --verbose</p><p>Dashboard（local loopback）：http://127.0.0.1:18789/</p><h2>配对+连接你的第一个聊天界面</h2><h3>WhatsApp（QR登录）</h3><p>运行openclaw channels login，然后通过WhatsApp→设置→链接设备扫描</p><h3>Telegram/Discord</h3><p>向导可以为你写入令牌/配置</p><h2>私信安全（配对审批）</h2><p>默认姿态：未知私信会获得一个短代码，消息在批准之前不会被处理</p><p>批准配对：openclaw pairing list whatsapp && openclaw pairing approve whatsapp <code></p>
