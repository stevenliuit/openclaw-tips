<h2>安全性</h2><h2>快速检查：openclaw security audit</h2><p>定期运行此命令（尤其是在更改配置或暴露网络接口之后）：</p><p>openclaw security audit</p><p>openclaw security audit --deep</p><p>openclaw security audit --fix</p><h2>审计检查内容（高层概述）</h2><ul><li><strong>入站访问</strong>（私信策略、群组策略、白名单）：陌生人能否触发机器人？</li><li><strong>工具影响范围</strong>（提权工具+开放房间）：提示词注入是否可能转化为shell/文件/网络操作？</li><li><strong>网络暴露</strong>（Gateway绑定、Tailscale认证、弱/短认证令牌）</li><li><strong>浏览器控制暴露</strong>（远程节点、中继端口、远程CDP端点）</li><li><strong>本地磁盘卫生</strong>（权限、符号链接、配置包含）</li></ul><h2>凭证存储映射</h2><ul><li><strong>WhatsApp</strong>：~/.openclaw/credentials/whatsapp/&lt;accountId&gt;/creds.json</li><li><strong>Telegram bot token</strong>：配置/环境变量或channels.telegram.tokenFile</li><li><strong>Discord bot token</strong>：配置/环境变量</li><li><strong>Slack tokens</strong>：配置/环境变量</li><li><strong>配对白名单</strong>：~/.openclaw/credentials/&lt;channel&gt;-allowFrom.json</li><li><strong>模型认证配置</strong>：~/.openclaw/agents/&lt;agentId&gt;/agent/auth-profiles.json</li><li><strong>旧版OAuth导入</strong>：~/.openclaw/credentials/oauth.json</li></ul><h2>安全审计清单</h2><ol><li>任何开放+启用工具的情况：首先锁定私信/群组，然后收紧工具策略/沙箱隔离</li><li>公共网络暴露：立即修复</li><li>浏览器控制远程暴露：将其视为操作员访问权限</li><li>权限：确保状态/配置/凭证/认证文件不是组/全局可读的</li><li>插件/扩展：只加载你明确信任的内容</li><li>模型选择：对于任何带有工具的机器人，优先使用现代的、经过指令强化的模型</li></ol><h2>私信访问模型（配对/白名单/开放/禁用）</h2><ul><li><strong>pairing</strong>（默认）：未知发送者会收到一个短配对码，机器人会忽略他们的消息直到获得批准</li><li><strong>allowlist</strong>：未知发送者被阻止</li><li><strong>open</strong>：允许任何人发私信（需要渠道白名单包含*）</li><li><strong>disabled</strong>：完全忽略入站私信</li></ul><h2>通过HTTP访问控制UI</h2><p>控制UI需要安全上下文（HTTPS或localhost）来生成设备身份。如果启用gateway.controlUi.allowInsecureAuth，UI会回退到仅令牌认证。</p><h2>威胁模型</h2><p>你的AI助手可以：执行任意shell命令、读写文件、访问网络服务、向任何人发送消息。给你发消息的人可以：试图欺骗你的AI做坏事、社会工程获取你的数据访问权限、探测基础设施详情</p><h2>核心概念：访问控制优先于智能</h2><ul><li><strong>身份优先</strong>：决定谁可以与机器人交谈</li><li><strong>范围其次</strong>：决定机器人被允许在哪里执行操作</li><li><strong>模型最后</strong>：假设模型可以被操纵；设计时让操纵的影响范围有限</li></ul><h2>提示词注入（是什么，为什么重要）</h2><p>提示词注入是指攻击者构造一条消息来操纵模型做不安全的事情。即使有强大的系统提示词，提示词注入也没有解决。实践中有帮助的是：</p><ul><li>保持入站私信锁定（配对/白名单）</li><li>在群组中优先使用提及门控</li><li>默认将链接、附件和粘贴的指令视为恶意的</li><li>在沙箱中运行敏感的工具执行</li><li>将高风险工具（exec、browser、web_fetch、web_search）限制给受信任的智能体或显式白名单</li></ul><h2>事件响应（如果你怀疑被入侵）</h2><h3>阻止影响范围</h3><ul><li>禁用提权工具（或停止Gateway网关）直到你了解发生了什么</li><li>锁定入站接口（私信策略、群组白名单、提及门控）</li></ul><h3>轮换秘密</h3><ul><li>轮换gateway.auth令牌/密码</li><li>轮换hooks.token并撤销任何可疑的节点配对</li><li>轮换提供商/API凭证</li></ul><h3>审查产物</h3><ul><li>检查Gateway网关日志和最近的会话/记录中是否有意外的工具调用</li><li>审查extensions/并移除任何你不完全信任的内容</li></ul><h3>重新运行审计</h3><p>openclaw security audit --deep并确认报告是干净的</p>
