<h2>前言</h2><p>我在树莓派上本地运行 OpenClaw——那里计算能力稀缺，它在我面前无响应的次数不少。但即使在受限硬件上，每次聊天轮次、每次记忆搜索、每次网络查找都在调用付费 API。账单起初很小，然后就不是了。我曾使用 qwen3-coder-480b 一两周，每天的成本飙升至高达 50 美元。</p><h2>假设</h2><p>OpenClaw 运行在你已经拥有或单独付费的硬件上——树莓派、家庭服务器或现有云实例。主机的计算成本不计入这里的每月约 20 美元中。</p><h2>谁应该关心</h2><p>两个非常不同的原因关心这个设置。</p><p><strong>如果你有 AWS 积分要消耗：</strong></p><p>来自 re:Invent、AWS Community Builder、AWS Activate 或客户计划的积分有有效期。在它们过期前，使用它们运行 AI 助手堆栈是最实际的利用闲置积分的方式——每月约 20 美元，100 积分可以覆盖 5 个月的全栈。如果你手头有几百美元且年底到期，这是它们失效前的生产性用途。</p><p><strong>如果你在公司有采购或合规要求：</strong></p><p>每个新的 SaaS 供应商都是一次 TPRM 练习。OpenAI 用于嵌入、Perplexity 用于网络搜索、Anthropic 用于 Claude——每一个都是单独的供应商评估、单独的 DPA，以及与你安全团队的单独对话。对于 FSI 和受监管行业，这不仅仅是开销——它可能是一个障碍。</p><p>AWS 可能已经在你的供应商登记册中。整合到 Bedrock 意味着单一账单、更少的第三方关系要管理，以及你控制的数据驻留。对于任何接触银行、保险或医疗保健中客户数据的东西，这是快速内部审批和 3 个月采购周期之间的区别。</p><h2>前置条件</h2><ul><li><strong>AWS 账户</strong>——在 us-east-1（或另一个美国区域）启用 Bedrock 访问</li><li><strong>AWS 凭证</strong>—— Bedrock API 密钥是最简单的选择（如果你的账户支持）。否则，长期 IAM 访问密钥/密钥对也很好，比 SSO 更容易管理。IAM Identity Center 仅在 Q Developer Pro 层需要。</li><li><strong>Python 3.10+</strong>——由 kiro-gateway、LiteLLM 和 Nova grounding 代理使用</li><li><strong>Amazon Q Developer Pro 订阅</strong>（19 美元/用户/月，积分可用）——第 1 层所需（kiro-gateway）。Kiro Pro、Pro+ 或 Power 计划也可以，但有积分和超额费用——Q Developer Pro 是更好的交易。</li></ul><h2>OpenClaw 中什么真正花钱？</h2><p>在寻找解决方案之前，了解支出去向很有帮助。OpenClaw 有四个不同的成本中心：</p><p><strong>1. 主模型（LLM）</strong></p><p>每个聊天轮次、每个智能体动作、每个工具调用——都通过你的主 LLM 路由。这是最大的可变成本。在繁忙的一天，它会快速累积。</p><p><strong>2. 记忆搜索（嵌入）</strong></p><p>OpenClaw 的 memory_search 工具将你的记忆文件转换为向量嵌入并语义查询它们。每次搜索 = 一次嵌入 API 调用。每次调用成本低，但它在后台持续运行。</p><p><strong>3. 网络搜索</strong></p><p>web_search 工具调用 Perplexity 或 Brave API。Perplexity 对付费计划按查询收费；Brave 每月给你 5 美元免费，之后收费。</p><p><strong>4. 浏览器自动化</strong></p><p>browser 工具为网页抓取、表单填写和截图启动 Chromium 实例。在低计算机器上运行完整浏览器（树莓派、t4g.small）很重——云浏览器选项按会话收费。</p><p>就这样。四层。目标：将可变成本降至零。</p><h2>我的配置：全部 4 层使用 AWS 积分</h2><h3>第 1 层：主模型 + 图像分析——Kiro CLI——由 AWS 积分覆盖</h3><p>Amazon Q Developer Pro 和 Kiro Pro 之间的关键区别是计费模式。Kiro Pro 是基于积分的——每月 1,000 积分，超出付费。Amazon Q Developer Pro 是固定月订阅：</p><p><strong>19 美元/用户/月，无按token计费，无意外超额。</strong></p><p><strong>注意：</strong>Amazon Q Developer Pro 现在是 Kiro 生态系统中的遗留计划。AWS 已停止允许新的 Builder ID 订阅 Q Developer Pro——新用户只能通过 Kiro 计划订阅。Q Pro 上未记录的使用限制可能是 AWS 做此转变的原因。如果你已经在 Q Developer Pro 上，你保留访问权限，它仍然是 OpenClaw 的更好交易。</p><h3>kiro-gateway：桥梁</h3><p>kiro-gateway——由 @Jwadow 构建——包装 Kiro CLI 并暴露 OpenAI 兼容和 Anthropic 兼容的 API 端点。OpenClaw 像连接任何其他提供商一样连接它。</p><h3>第 2 层：记忆搜索——Bedrock 嵌入——由 AWS 积分覆盖</h3><p>OpenClaw 的 memory_search 需要嵌入模型。Amazon Nova 多模态嵌入每 1K tokens 成本约 0.00014 美元——每次查询几分钱，并由 AWS 积分覆盖。</p><h3>第 3 层：网络搜索——Nova Grounding 代理——由 AWS 积分覆盖</h3><p>我构建了 bedrock-web-search-proxy——一个 FastAPI 包装器，使 Bedrock Nova Grounding 看起来像 Perplexity Sonar API。不需要 Perplexity 或 Brave API 密钥。完全在 AWS 积分上运行。</p><h3>第 4 层：云浏览器——Bedrock AgentCore——由 AWS 积分覆盖</h3><p>agent-browser 由 Vercel Labs 构建，带有由 Pahud Hsieh（@pahudnet）贡献的 AgentCore 提供商。浏览器在 AWS 中运行——不需要本地 Chromium。在低计算实例上特别有用。</p><h2>成本计算</h2><p>没有这个设置，仅 Claude Sonnet 在标准 Bedrock 定价下运行约 <strong>1,000 美元/月</strong>——基于我自己会话中的真实 token 使用。OpenClaw 的大上下文窗口（每次轮次加载记忆文件、系统提示、工具结果）意味着 token 账单快速累积。</p><p>使用此设置的完整堆栈运行约 <strong>每月 20 美元</strong>：</p><ul><li><strong>19 美元/月</strong>—— Amazon Q Developer Pro（固定费率，覆盖所有 LLM 调用）</li><li><strong>≤1 美元/月</strong>—— Bedrock 嵌入用于记忆搜索（Nova 2 为 0.00014 美元/1K tokens）</li></ul><p>网络搜索和浏览器自动化由 AWS 积分覆盖——没有单独的行项目。</p><p>使用 <strong>100 美元 AWS 积分</strong>，你可以覆盖约 <strong>5 个月</strong>的全栈。Q Developer Pro 订阅和 Bedrock 嵌入都有积分资格——如果你是 AWS Community Builder，那每年 500 美元的配额足以覆盖它。</p>
