<h2>设置</h2><h2>太长不看</h2><p><strong>个性化设置存放在仓库之外</strong>：~/.openclaw/workspace（工作区）+ ~/.openclaw/openclaw.json（配置）</p><p><strong>稳定工作流</strong>：安装macOS应用；让它运行内置的Gateway网关。</p><p><strong>前沿工作流</strong>：通过pnpm gateway:watch自己运行Gateway网关，然后让macOS应用以本地模式连接。</p><h2>先决条件（从源码）</h2><ul><li>Node.js >= 22</li><li>pnpm</li><li>Docker（可选；仅用于容器化设置/e2e）</li></ul><h2>个性化策略（让更新不会造成问题）</h2><p>如果你想要100%为我定制并且易于更新，将你的自定义内容保存在：配置：~/.openclaw/openclaw.json、工作区：~/.openclaw/workspace（Skills、提示、记忆；将其设为私有git仓库）</p><h2>稳定工作流（macOS应用优先）</h2><ol><li>安装并启动OpenClaw.app（菜单栏）</li><li>完成新手引导/权限检查清单（TCC提示）</li><li>确保Gateway网关是本地并正在运行（应用管理它）</li><li>链接表面（示例：WhatsApp）：openclaw channels login</li><li>完整性检查：openclaw health</li></ol><h2>前沿工作流（在终端中运行Gateway网关）</h2><p>目标：开发TypeScript Gateway网关，获得热重载，保持macOS应用UI连接。</p><h3>0)（可选）也从源码运行macOS应用</h3><p>./scripts/restart-mac.sh</p><h3>1) 启动开发Gateway网关</h3><p>pnpm install && pnpm gateway:watch</p><h3>2) 将macOS应用指向你正在运行的Gateway网关</h3><p>在OpenClaw.app中：连接模式：本地</p><h3>3) 验证</h3><p>应用内Gateway网关状态应显示Using existing gateway...</p><h2>常见陷阱</h2><ul><li><strong>端口错误</strong>：Gateway网关WS默认为ws://127.0.0.1:18789；保持应用 + CLI在同一端口上</li><li><strong>状态存储位置</strong>：凭证：~/.openclaw/credentials/、会话：~/.openclaw/agents/&lt;agentId&gt;/sessions/、日志：/tmp/openclaw/</li></ul><h2>凭证存储映射</h2><ul><li><strong>WhatsApp</strong>：~/.openclaw/credentials/whatsapp/&lt;accountId&gt;/creds.json</li><li><strong>Telegram bot token</strong>：配置/环境变量或channels.telegram.tokenFile</li><li><strong>Discord bot token</strong>：配置/环境变量</li><li><strong>Slack tokens</strong>：配置/环境变量（channels.slack.*）</li><li><strong>配对允许列表</strong>：~/.openclaw/credentials/&lt;channel&gt;-allowFrom.json</li><li><strong>模型认证配置文件</strong>：~/.openclaw/agents/&lt;agentId&gt;/agent/auth-profiles.json</li><li><strong>旧版OAuth导入</strong>：~/.openclaw/credentials/oauth.json</li></ul><h2>更新（不破坏你的设置）</h2><p>将~/.openclaw/workspace和~/.openclaw/保持为你的东西；不要将个人提示/配置放入openclaw仓库。更新源码：git pull + pnpm install + 继续使用pnpm gateway:watch。</p><h2>Linux（systemd用户服务）</h2><p>Linux安装使用systemd用户服务。默认情况下，systemd在注销/空闲时停止用户服务。运行：sudo loginctl enable-linger $USER</p>
