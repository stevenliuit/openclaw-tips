# GPT API 进阶用法：结构化输出、流式响应与 Function Calling 最佳实践

大多数开发者对 GPT API 的使用停留在简单对话层面。殊不知，API 提供了多项高级特性，可以显著提升应用的智能化程度和用户体验。本文将深入解析这些进阶用法及其最佳实践。

## 结构化输出：JSON Mode 与 Response Format

GPT 模型生成的内容默认是自由文本，解析成本高且不稳定。GPT API 的 JSON Mode 通过约束模型输出为合法 JSON 格式，解决了这一问题。启用方式是在请求中设置 `response_format: { type: "json_object" }`。需要注意的是，JSON Mode 只保证输出是合法 JSON，不验证内容是否匹配特定 Schema。

对于需要严格格式控制的场景，OpenAI 推出了更强大的 Structured Outputs 特性。通过定义完整的 JSON Schema，模型被强制约束输出必须符合 Schema 定义。这对于构建需要解析后继续处理的管道尤为重要——例如从用户描述中提取结构化订单信息，后续需要交给订单系统处理。

使用 Structured Outputs 时，Schema 定义应当遵循以下原则：字段名使用 camelCase 或 snake_case 保持一致；使用 enum 限制可选值范围；添加 description 字段帮助模型理解字段含义；对于可选字段设置合理的默认值。

## 流式响应：提升用户体验的关键

流式响应（Streaming）是改善用户感知延迟的核心技术。当模型生成一段长文本时，非流式模式下用户需要等待数秒才能看到完整回复；流式模式下，内容以小块形式实时返回，用户几乎可以即时看到生成进度。

实现流式响应的技术基础是 Server-Sent Events（SSE）。API 返回的流式响应本质上是一个 HTTP 长连接，数据以 `data: {...}` 格式的消息体返回。前端需要解析这些消息，提取文本增量并更新 UI。主流前端框架都提供了现成的流式响应处理库。

流式响应的应用场景选择也很重要。对于需要完整内容才能进行下一步操作的场景（如需要 LLM 输出作为另一个系统的输入），流式反而增加复杂度。但对于对话、摘要、代码生成等以人类为最终用户的场景，流式响应能显著提升体验。

## Function Calling：让 LLM 调用外部工具

Function Calling 是 GPT API 最强大的扩展能力之一。通过在请求中定义函数规格（名称、参数 Schema、描述），模型能够理解何时需要调用外部工具，并自动生成符合规范的参数。

一个设计良好的 Function Calling 系统需要注意以下几点。函数命名应当清晰表达功能，避免歧义。参数 Schema 应当严格定义类型和约束，使用 enum 限制可选值范围，添加必要的 description 帮助模型理解。函数描述是模型决定是否调用的关键依据，应当明确说明函数的适用场景和返回值含义。

实际的架构实现中，建议使用「路由 + 执行」模式。模型返回的函数调用请求首先进入调度器，调度器根据函数名路由到对应的处理函数。处理函数应当是幂等的，支持重试，且执行时间不宜过长——如果需要长时间处理，考虑使用异步模式，立即返回任务 ID。

## 上下文窗口管理与 Token 优化

GPT 模型有固定的上下文窗口限制（当前主流模型为 128k tokens），超出限制的内容会被截断。高效的上下文管理直接影响应用的效果和成本。

Token 优化从 Prompt 工程开始。冗余的 instruction、重复的示例、过长的系统提示都会消耗宝贵的上下文空间。建议定期审视 Prompt，删除无效内容，压缩表述。此外，很多场景可以使用更小的模型替代通用大模型，在效果损失可接受范围内大幅降低 Token 消耗。

对话历史的压缩也是重要议题。简单的截断策略可能丢失关键上下文。更好的方案是使用摘要模型定期将对话历史压缩为摘要，或采用「重要信息提取 + 滚动窗口」策略，只保留最近 N 轮对话加上提取的关键信息。

## 错误处理与重试策略

API 调用不可避免会遇到网络异常、限流、模型过载等问题。健壮的错误处理机制应当区分不同错误类型并采取相应策略：网络超时适合重试；限流错误（429）应当等待后重试且增加退避时间；服务器错误（500-503）短期可能恢复，适合重试；认证错误（401）需要检查 API Key 有效性，不应盲目重试。

指数退避是处理限流的推荐策略。首次失败后等待 1 秒重试，若仍失败等待 2 秒，以此类推，最大等待时间通常设为 60 秒。部分 API 提供商会在响应头中携带 `Retry-After` 信息，应当优先使用该值作为等待时间。

GPT API 的进阶用法为构建智能化应用提供了强大的基础设施支持。熟练掌握这些特性并遵循最佳实践，能够显著提升应用的智能水平和工程质量。
